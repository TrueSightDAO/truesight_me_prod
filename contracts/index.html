TrueSight DAO Protocol

Smart Contracts

Every action in the TrueSight DAO is an RSA-signed smart contract. Each contract is a cryptographically signed event submitted to Edgar — the DAO's execution layer — which verifies the signature, applies state transitions, and records the result on the ledger and TrueChain audit trail.

30 Smart Contracts
6 Categories
RSA-2048 Signature Algorithm
SHA-256 Hash Function

Protocol Architecture

The four-layer architecture of the TrueSight DAO protocol. Every signed event flows from the DApp through the smart contract layer to Edgar, which verifies and records it.

How Signing Works

1

Generate Keypair

Create an RSA-2048 keypair in the DApp or via the CLI. Your public key becomes your DAO identity.

2

Register Identity

Submit an [EMAIL REGISTERED EVENT] to bind your public key to your email. Verify via the link sent to your inbox.

3

Sign a Contract

Choose an event type, fill in the fields, and sign the canonical payload with your private key.

4

Submit to Edgar

POST the signed payload to edgar.truesight.me. Edgar verifies the signature and executes the state transition.

Identity & Registration

6 contracts

Email Registered Event [EMAIL REGISTERED EVENT]

Register a new email address with the DAO. The first step to creating a DAO identity — binds an email to a public key.

Anyone Required: Email

Email Verification Event [EMAIL VERIFICATION EVENT]

Verify ownership of an email address by signing a challenge. Completes the identity registration flow.

Anyone (with pending registration) Required: Email, Verification Key

Contributor Add Event [CONTRIBUTOR ADD EVENT]

Onboard a new contributor to the DAO. Records their name, email, and role in the contributor registry.

Governor Required: Contributor Name, Email, Role

DApp Permission Change Event [DAPP PERMISSION CHANGE EVENT]

Grant or revoke DApp access permissions for a contributor. Controls who can sign contracts via the web interface.

Governor Required: Contributor Email, Permission Level

Withdraw Voting Rights Event [WITHDRAW VOTING RIGHTS EVENT]

Remove a contributor's voting rights. A governance action for managing DAO membership.

Governor Required: Contributor Email

Program Registration Event [PROGRAM REGISTRATION EVENT]

Register a new program or initiative within the DAO ecosystem (e.g. credentialing cohort, community challenge).

Governor Required: Program Name, Program Type, Start Date

Inventory & Supply Chain

5 contracts

QR Code Registration [QR CODE REGISTRATION]

Register a new batch of QR codes for cacao bags. Each QR code uniquely identifies a bag in the supply chain.

Governor / Inventory Manager Required: QR Code(s), Batch ID, Product Type
Variants

Single Registration (one QR code)

Register one QR code at a time for individual bag tracking.

Batch Registration (bulk QR codes)

Register multiple QR codes in a single transaction for efficiency.

Inventory Movement [INVENTORY MOVEMENT]

Transfer custody of cacao bags between known supply-chain holders. QR status remains IN INVENTORY throughout.

Inventory Manager Required: Manager Name, Recipient Name, QR Code(s), Quantity
Variants

Single Transfer (one bag)

Move one bag from one holder to another.

Batch Transfer (multiple bags)

Move multiple bags in a single transaction.

Consignment Transfer (to retailer on consignment)

Transfer bags to a retailer with consignment terms. Bags remain DAO property until sold.

Sales Event [SALES EVENT]

Record a retail sale of cacao to an end customer. QR status becomes SOLD. Triggers revenue tracking.

Seller / Retailer Required: Item, Sales Price, Sold By, Cash Proceeds Collected By, Owner Email
Variants

Single Sale (one bag)

Sell one bag to a customer.

Bulk Sale (multiple bags)

Sell multiple bags in a single transaction.

Consignment Settlement (sale from consignment stock)

Settle a sale from bags previously placed on consignment. Triggers revenue split between DAO and retailer.

Asset Receipt [ASSET RECEIPT]

Confirm receipt of physical assets (cacao bags, materials) into inventory. Creates the initial inventory record.

Inventory Manager Required: Asset Type, Quantity, Received By, Source

Currency Conversion [CURRENCY CONVERSION]

Record a currency conversion event (e.g. USD to BRL) for financial tracking across the supply chain.

Treasury Manager Required: From Currency, To Currency, Amount, Exchange Rate

Contribution & Finance

8 contracts

Contribution Event [CONTRIBUTION EVENT]

Record time or value contributed to the DAO. The primary mechanism for earning TDG governance tokens.

Any Contributor Required: Type (Time/USD), Amount, Contributor(s), Description
Variants

Time Contribution (minutes worked)

Log time spent on DAO work. Converted to TDG at the current rate.

USD Contribution (monetary value)

Record a USD-denominated contribution (e.g. services rendered at market rate).

AI Agent Contribution (automated)

Sophia and other AI agents submit this variant to log autonomous work.

Capital Injection Event [CAPITAL INJECTION EVENT]

Record a capital injection into the DAO treasury. Tracks funding sources and amounts.

Treasury Manager / Governor Required: Amount, Source, Currency, Purpose

DAO Expenses Event [DAO EXPENSES EVENT]

Record an expense paid from the DAO treasury. Tracks operational costs and reimbursements.

Treasury Manager Required: Amount, Category, Payee, Description

Mint Donation Event [MINT DONATION EVENT]

Mint a donation receipt as an on-chain record. Used for charitable contributions and tax receipting.

Treasury Manager Required: Donor Name, Amount, Currency, Project

Payment Event [PAYMENT EVENT]

Record a payment made or received by the DAO. Tracks all financial transactions.

Treasury Manager Required: Amount, Payee/Payer, Currency, Reference

Notarize Event [NOTARIZE EVENT]

Notarize a document or record by hashing it and storing the hash on-chain. Provides timestamped proof of existence.

Any Contributor Required: Document Hash, Description, Timestamp

Link Attestations to Trees [LINK ATTESTATIONS TO TREES]

Link attestation records to specific trees in the Sunmint reforestation program. Proves impact per tree.

Program Manager Required: Tree ID(s), Attestation Hash, Program

Repackaging Planner [REPACKAGING PLANNER]

Plan and record the repackaging of bulk cacao into smaller retail units. Tracks bag conversions.

Inventory Manager Required: Source QR Code(s), Destination Quantity, Package Size
